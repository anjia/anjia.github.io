jQuery.ajax(url [,settings])
执行一个异步 HTTP（Ajax）请求

url

类型：String
一个字符串，包含请求被发送到的 URL

settings

类型：PlainObject
一组配置 Ajax 请求的 键/值 对，所有设置都是可选的。任何选项的默认值都可以通过 $.ajaxSetup() 来设置。

API 中一共提到了 34 个设置选项，为了方便理解，这里将它们归类介绍下。

request 的方法

method

类型：String
默认值：’GET’
请求使用的 HTTP 方法（比如. “POST” “GET” “PUT”） [在 v1.9.0 中新增的]

type

类型：String
默认值 ‘GET’
method 的别名。如果你使用 jQuery 1.9.0 之前的版本，那应该使用 type。

data

类型：PlainObject 或 String 或 Array
发送到服务器的数据。当是 GET 请求时，它会被追加到 url 的后面。如果不是字符串类型，它会被转化成查询字符串，processData 选项可以防止这种自动处理。对象必须是 键/值 对，如果值是 Array，jQuery 会用相同的 key 去序列化多个值，基于 traditional 选项设置的值。

类型：Boolean
默认值：true
默认情况，在 data 中放的数据会被处理成一个对象（从技术上来讲，除了 String），并转换成查询字符串以适应默认的内容类型（content type) “application/x-www-form-urlencoded”。如果你想发送一个 DOM 文档，或者其他非加工的数据，设置这个选项为 false。

contentType

类型：Boolean 或 String
默认值：’application/x-www-form-urlencoded; charset=UTF-8’
当向服务器发送数据时，使用这个内容类型，默认是 “application/x-www-form-urlencoded; charset=UTF-8”，大多数时候都能满足。
如果明确给 $.ajax() 传递内容类型，那么它总会发送一个请求到服务器（即使没有数据要发送）。在 jQuery 1.6 中，可以设置成 false 告诉 jQuery 不设置任何内容类型头。
Note：W3C XMLHttpRequest 规范规定编码总是 UTF-8，指定其他的字符集并不会强制浏览器去改变编码。
Note：对于跨域请求，把内容类型设置成 application/x-www-form-urlencoded, multipart/form-data, 和 text/plain 以外的任何值，将会触发浏览器发送一个 preflight OPTIONS 请求给服务器。

异步

async

类型：Boolean
默认值：true
默认情况下，所有的请求会被异步发送（也就是说，它默认设置是 true）。如果你需要同步请求，那就把这个选项设置成 false。
跨域请求 和 dataType:”jsonp” 的请求不支持同步选项。注意，同步请求可能会暂时锁住浏览器，当请求还在处理的时候会导致浏览器禁用任何操作。
在 jQuery 1.8 版本中，jqXHR($.Deferred) 的 async:false 已经被废弃了；你必须使用 success/error/complete 的回调选项替代 jqXHR 提供的响应方法，诸如 jqXHR.done() 还有被废弃的 jqXHR.success()。

跨域

jsonp

类型：String
在 JSONP 请求中覆盖 callback 函数。
这个值将会被使用 替代 url 中的查询字符串中的 ‘callback=?’部分中的’callback’。
因此 {jsonp:’onJSONPLoad’} 将会变成 ‘onJSONPLoad=?’ 被传送到服务器。
在 jQuery 1.5 中，设置 jsonp 选项为 false 将会组织 jQuery 添加 “?callback” 字符串给 URL 或者尝试变成 “=?”。在这种情况下，你也应该地设置 jsonpCallback。比如 {jsonp: false, jsonpCallback: “callbackName”}

jsonpCallback

类型：String 或 function()
为 JSONP 请求明确指定一个回调函数名。这个值会被使用 替代 jQuery 自动生成的随机的名字。
最好让 jQuery 生成一个唯一的名称，因为它将会变得更容易 管理请求同时提供回调和错误处理。【？也就是鼓励不指定喽？】
你可能想要指定一个回调名字 当你想要更好的使用浏览器缓存的GET请求时【？也就是指定了，就不唯一了，就有可能走缓存。如果每次都想直达服务器，则就不指定】
在 jQuery 1.5 中，你也可以给这个选项设置一个函数，在这种情况下 jsonpCallback 的值，被设置成那个函数的返回值

类型

dataType

类型：String
默认值：智能推测（xml, json, script, 或者 html）
你期望从服务器返回的数据类型。
如果没有指定，jQuery将会尝试infer它 基于响应中的MIME类型（
一个XML MIME类型将产生XML，在 1.4 JSON将会产生一个JS对象，
在 1.4 脚本将会执行脚本，其他的都会返回一个字符串）。
可选的类型（结果会作为第一个参数传给成功时的回调）都有：【这个类型对应的结果~ 而不是说这个类型。soga，连起来了】
“xml”: 返回一个XML文档，可以通过jQuery处理
“html”: 返回HTML作为plain text，包含的script标签 会被执行 当插入到DOM中时
“script”: 评估响应作为JS 并返回它作为plain text。
禁用缓存通过在URL添加一个查询字符串参数 =[TIMESTAMP]，除非 cache 选项被设置成了 true。
注意：这将会把POSTs变成GETs为了远程域的请求。【？是禁用缓存会这样吗？】
“json”: 评估响应作为JSON，返回一个JS对象。跨域 “json” 请求会被转化成 “jsonp” 除非请求中
包含 jsonp:false 在它的请求选项中。
JSON数据会被解析 用严格的方式【？是必须双引号吧~】 任何malformed的JSON被拒绝 会抛出一个解析错误。
jQuery 1.9，一个空的响应也会被拒绝；服务器应该返回 null 或者 {}。
（查看 json.org 更多信息关于适当的JSON格式）
“jsonp”: 使用JSONP加载在一个JSON块。
添加一个额外的 “?callback=?” 给你的URL的末尾，以指定回调。
禁用缓存通过添加一个查询字符串参数 “=[TIMESTAMP]” 给URL，除非 cache 选项被设置成了 true。
“text”: 一个plain text 字符串。

多个空格分隔的值：
jQuery 1.5，jQuery可以把一个dataType从 它接收到的在Content-Type头 转换成 你所要求的。
比如：如果你想 text 响应当成XML来处理，使用 “text xml”给dataType。
你也可以制造一个JSONP请求，当接收到它的时候变成text，这会阻止jQuery as XML:
“jsonp text xml”。
同样的，一个简写的字符串 比如 “jsonp xml” 将会首先试图把jsonp转成xml，如果失败了，
将会把jsonp转成text，再从text转成xml。

accepts

类型：PlainObject
默认值：取决于 DataType
在请求头中的content type，告诉服务器它将会接收什么类型的响应。

回调及相关

success

类型：Function(Anything data, String textStatus, jqXHR jqXHR)
如果请求成功就会回调这个函数。他有三个参数:
从服务器返回的data，格式化 通过 dataType 参数 或者 dataFilter回调函数，如果指定的话。
一个描述状态的字符串
jqXHR（在jQuery 1.4.x，XMLHttpRequest）对象。
在jQuery 1.5 中，成功的设置可以接受一个函数数组，每一个函数都会被按序会回调。
点击查看 Ajax Event。

dataFilter

类型：Function(String data, String type) => Anything
函数用来处理 XMLHttpRequest 原始的响应数据。
这是一个预过滤函数 去 sanitize 原始的响应数据。你应该返回 sanitized 数据。
这个函数接收两个参数：
从服务器返回的原始的数据 和 ‘dataType’ 参数

error

类型：Function(jqXHR jqXHR, String textStatus, String errorThrown)
如果请求失败了，就会回调这个函数。
这个函数接收三个参数：
jqXHR (jQuery 1.4.x 中是 XMLHttpRequest)对象
一个描述错误类型字符串，
一个可选的异常的对象，如果发生了异常。
第二个参数的值除了 null 外，还可以是 “timeout” “error” “abort” “parsererror”。
当发生了HTTP错误时，errorThrown 会收到HTTP状态的文本protion，比如 “Not Found” 或者
“Internal Server Error”。
在jQuery 1.5 中，error 设置可以接受一个函数数组。每一个函数将会被按顺序回调。
注意：这个处理器不会被回调 跨域脚本 和 跨域JSONP请求。
点击查看 Ajax Event。

complete

类型：Function(jqXHR jqXHR, String textStatus)
当请求完成了的时候函数会被调用（在 success 和 error 回调执行完之后）。
函数得到传过来的两个参数：
jqXHR（jQuery 1.4.x 中是 XMLHttpRequest）对象
和一个字符串 请求状态的分类（”success” “notmodified” “nocontent” “error” “timeout” “abort”
“parsererror” ）。
在 jQuery 1.5 中，complete 可以接受一个函数数组。每一个函数将会被按序回调。
点击查看 Ajax Event。

cache

类型：Boolean
默认值：true, 当 dataType是’script’和’jsonp’时默认是 false
如果把它设置成了false，它会强制请求页面不要被浏览器缓存 【那万一是中间的代理呢？】
注意：把cache设置成 false 只能正确的工作在 HEAD和GET请求。
它的工作方式是 给GET参数后追加是一个时间戳 “_={timestamp}”。
其他类型的请求的时候就不会用到这个参数，
IE8 中的是个例外，当一个POST is made to a URL 已经被一个GET请求过了。
